# progress.md

## Project: Small-Model Tool-Use Reminder Harness

## Current Goal

Build backend + execution harness for small-model tool use in reminder domain.

## Status (2026-05-10)

Phase-1 implemented and verified:

- Python project skeleton created (`app/`, `tests/`, `scripts/`, `data/`).
- Reminder backend service implemented:
  - `create_reminder`
  - `query_reminder`
  - `update_reminder`
  - `delete_reminder`
- Local JSON storage implemented (`data/reminders.json`).
- Structured JSON statuses implemented across tool results:
  - `success`
  - `missing_fields`
  - `not_found`
  - `ambiguous`
  - `error`
- OpenAI-compatible tool schemas implemented in canonical registry (`app/tool_registry.py`).
- Tool executor implemented (`app/tool_executor.py`):
  - accepts `assistant.tool_calls`
  - parses JSON args
  - validates required fields via canonical schema
  - executes backend tool
  - returns `role=tool` messages
- Runtime loop implemented (`app/runtime_loop.py`) with max rounds to prevent infinite loops:
  - assistant emits tool call
  - runtime pauses and executes tool
  - runtime appends tool message
  - model is called again for final answer

## Update (local popup reminder watcher)

Added a minimal local popup reminder runtime for existing `data/reminders.json` records.

New files:

- `app/reminder_notifier.py`
- `scripts/run_reminder_notifier.py`
- `tests/test_reminder_notifier.py`

Behavior:

- Polls reminders from JSON storage on an interval.
- Triggers popup only for reminders where:
  - `status == "active"`
  - `scheduled_time <= now`
  - no `notified_at` field yet
- After popup, writes `notified_at` and updates `updated_at` to avoid repeated popups.
- Supports one-shot mode for quick verification.

## Update (assistant follow-up generation)

Improved `scripts/export_training_data_llm.py` to avoid hardcoded assistant follow-up lines:

- Added LLM-based follow-up generation using real tool payload:
  - `DataBuilder(..., followup_client=client)`
  - after tool execution, assistant text is generated by LLM from:
    - original user text
    - tool name
    - real backend payload
- Added `OpenAICompatibleClient.chat_object(...)` helper for structured JSON response.
- Kept deterministic fallback templates only as fail-safe when follow-up LLM call fails.

This keeps tool outputs real (from local executor) while making assistant phrasing non-rigid.
